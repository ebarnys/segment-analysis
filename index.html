Identified gaps for Segment.com analysis
FREE_EXPIRED – almost no coverage
Users after day 365 receive no communication. Missing reactivation flow: "come back, a lot has changed" with a concrete offer or discount.
NOTIF-PAYMENT-PROBLEM – NOT SENT
The failed payment template exists (ID 80) but is never sent. Payment recovery email is typically one of the highest-ROI touchpoints in the entire funnel.
Days 15–45 – communication gap
FREE15 users receive almost nothing between the FREE1 transition (day 15) and NOTIF-50DISCOUNT (day 45). 30 days of silence = lost conversion momentum.
Women – different flow, but not reflected
TIP1–TIP4 and NOTIF-VIP-WELCOME are explicitly "not women", but women have no dedicated onboarding sequence. They likely receive inappropriate generic emails.
Post-VIP cancel – survey only
After VIP cancellation only NOTIF-SURVEY-CANCEL is sent. Missing win-back flow: offer of a cheaper plan, downgrade option, or timed reactivation emails.
Birthday discount – underrated touchpoint
NOTIF-20DISCOUNT-BRD is Low priority, despite personalized birthday emails with limited-time offers having above-average CTR and conversion in the dating sector.

Identified gaps – Overlay layer
Missing overlay for failed payment
"Failed Automatic coin refill" exists only as a mention (deleted card), but there is no overlay for VIP payment failure. Meanwhile, NOTIF-PAYMENT-PROBLEM (ID 80) exists in the email layer and is also never sent – overlay + email recovery flow is completely missing.
No proactive conversion overlay for FREE15 (days 15–45)
The email layer already has an identified gap for days 15–45 without communication. The overlay layer also has no proactive trigger – only reacts to user actions (feature gates). Missing: a timed overlay (e.g. day 20, 30) with a limited offer.
Post-VIP cancel – no win-back overlay
No overlay exists after VIP Membership cancellation. The email layer only sends a survey (NOTIF-SURVEY-CANCEL). Ovoverlay on first login after VIP expiry offers no reactivation incentive or cheaper plan.
Welcome message – generic for all users, no personalisation
The Welcome message is shown to all users identically on every login. For FREE users after day 15, 30 or 45 it could contain a different CTA (discount, urgency). For VIP users after expiry it should contain a win-back offer.
Cross-sell overlay without A/B testing framework
The cross-sell promo overlay is controlled only by a CoolAdmin timer for all users. Missing: segmentation by activity, gegender or lifecycle stage. The overlay is shown the same way to a user on day 3 and day 200.
No reactivation overlay for FREE_EXPIRED (after day 365)
FREE_EXPIRED users have no dedicated overlay. On return to the member area (via magic link from email) they only see the generic Welcome message with no specific reactivation offer.
Push opt-in overlays – 3 variants, unclear coordination
There are 3 separate push overlays (chat, hotlist, match). It is unclear whether they show only once or repeatedly. Without coordination all three could appear in a single session – causing push fatigue for new users.
Disabled overlays without documented reason
5 overlays are marked as OFF (Coin Paymentgate, Push confirmation, migrations, Coin system). Missing documentation on why they were disabled and whether they are candidates for reactivation. Coin system has only the note "Ask Tomas".

Each audience is defined by a combination of behavioral signals — not a single event. The goal is to identify where each user stands in their intent-to-pay journey and communicate accordingly. Audiences 1–5 form a linear conversion funnel. Audiences 6–8 are cross-cutting profiles.

On the platform but showing zero interest in any paid features. Only passive browsing.
Low intent
Conditions – all must be true
·Status: FREE, registered 14+ days ago
·Never triggered vip_payment_gate_viewed or coins_payment_gate_viewed
·0 paywall overlay events fired (*_vip_overlay_shown)
·Engagement score low — only standard events (browsing, filtering)

·vip_payment_gate_viewed count = 1 (visited once, didn't return)
·Paywall hit in max 1 feature
·No failed payments, engagement score medium

Show the value of VIP through content (what others get), offer a 7-day trial or 2 free Live Chat messages. Low-pressure.

Actively using the platform, starting to hit FREE limits. Visited the payment gate multiple times — still window shopping.

💬 Message types

There are 5 distinct message types in the system. Regular, predefined, and gift messages are user-initiated and count (or not) toward the daily limit. Livechat and robot messages are platform-side mechanics.

✏️ Regular message
User writes their own text from scratch — the most personal type. Counts toward the daily limit.
Response
A: Moderator writes manual reply  |  C: System picks from the universal response pool. Split is 50 / 50.
📋 Predefined message
User selects one of the offered pre-written message options from a list. Counts toward the daily limit.
Response
Always automatic — each predefined question is paired with multiple response variants; the system picks one instantly.
🎁 Gift message
User sends a virtual gift (paid with coins) and attaches a personal note. Does NOT count toward the daily limit.
Response
Always a moderator — the personal gesture deserves a personal reply. Never automated.
💬 Livechat message
Paid real-time chat. Available to VIP users only. Each message costs 10 coins. Does not count toward the daily message limit — it has its own coin-based economy.
Response
Handled by the Livechat Responder queue. Conversations sorted by whale tier — highest-value users answered first.
🤖 Robot message (profile robot)
System-initiated outreach. Each fake female profile has exactly one robot message inspired by that profile's persona. Sent to male users who have not been contacted yet. Lower engagement than Active Responder because it cannot reference the recipient's profile.
Trigger & initial burst
Starts ~2 minutes after the user completes TOI (Tour of Introduction / onboarding). User receives a platform notification in chat + an email.
Within the first 5–120 minutes, the user receives 4 robot messages from 4 different fake profiles.

Auto-reactivation
If a whale user hasn't sent a Livechat message for a set number of days, the system automatically sends them a message from their last active female profile — a short, inviting line to re-open the chat. The higher the whale tier, the more aggressively the auto-reactivation fires. Goal: keep high-value users engaged and spending.
